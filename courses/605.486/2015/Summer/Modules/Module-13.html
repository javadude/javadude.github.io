Module 13 - Notifications

605.486 - Mobile Application Development for the Android Platform

Overview

Activities actively interact with a user. They can readily communicate with the user via fields, Toasts and dialogs as part of the user's current workflow. However, background threads and Services (covered in a later module) perform processing that may be unrelated to the user's current task. Raising a Dialog or Toast would disrupt the user.

In this module, we'll see how to post Notifications to the Notification Bar that's usually visible at the top of most applications. These Notifications contain data and sometimes actions that the user can perform, and do not interrupt the current application. The user can interact with them at any time.

Objectives

By the end of this module, you will be able to:

  • Describe why Dialogs and Toasts should not be used from background threads or Services
  • Create and post Notifications to the Notification Bar
  • Customize Notifications using expansion styles (Jelly Bean and later) and actions.
