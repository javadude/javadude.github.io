JavaDude Annotations: Properties

[Back to main JavaDude Annotations page]

A property is defined by the presence of the following types of methods:

void setName(Type value);

boolean isName();

Type getName();

where Name represents the name of the property. These methods typically read/update a field in the class, but they are not required to do so.

The name of the property is the same as the method name, after removing the get/set/is and lowercasing the first letter. For example:

Method Property
void setFirstName(String value) firstName
String getAverageGrade() averageGrade
boolean isEmpty empty

Defining a Property

You define a property by adding an @Property annotation to your @Bean annotation:

package com.javadude.annotation.example;



import com.javadude.annotation.Bean;

import com.javadude.annotation.Property;



@Bean(properties = {

    @Property(name="name")

    }

)  

public class Person extends PersonGen {



}

The only value you're required to specify is the name of the property. By default, the annotation processor will generate a simple, non-bound, read/write, String property with public get/set/is methods. In this case, the generated code is

// CODE GENERATED BY JAVADUDE BEAN ANNOTATION PROCESSOR

//   -- DO NOT EDIT -- WILL BE REGENERATED!

package com.javadude.annotation.example;



public abstract class PersonGen  {

    public PersonGen() {

        ;

    }

    private java.lang.String name_;

    public java.lang.String getName() { return name_; }

    public void setName(java.lang.String value)  {

        name_ = value;

    }

    @Override

    public String toString() {

        return getClass().getName() + '[' + paramString() + ']';

    }

    protected String paramString() {

        return "name=" + name_;

    }

}
Simple Properties

Simple properties are properties that have a single-value. We'll look at multi-valued properties shortly. When defining a simple property, you can set the following attributes of the @Property annotation:

name
The name of the property to generate. This will result in generation of a private field in the generated superclass.
type
The type of the generated property. Specify this as a class instance. For example, Address.class, int.class (for the primitive type int), Integer.class (for the wrapper type Integer), String.class, etc. This attribute value defaults to String.class.
reader
The existence and protection level of the get/is method for the property. This can have the following values:
  • Access.NONE - do not generate a get/is method
  • Access.PROTECTED - the generated get/is method will be declared protected
  • Access.PACKAGE - the generated get/is method will be declared without an access modifier
  • Access.PUBLIC - the generated get/is method will be declared public
If not specified, this value will default to the reader attribute of the @Bean annotation (which can have the same values as the above).
writer
The existence and protection level of the set method for the property. This can have the following values:
  • Access.NONE - do not generate a set method
  • Access.PROTECTED - the generated set method will be declared protected
  • Access.PACKAGE - the generated set method will be declared without an access modifier
  • Access.PUBLIC - the generated set method will be declared public
If not specified, this value will default to the writer attribute of the @Bean annotation (which can have the same values as the above).
bound
true or false. If true, and generating a writer for the property, bound property support will be added (add/removePropertyChangeListener methods for tracking observers) and the setter method will contain code to fire a property change event.

Some examples:

package com.javadude.annotation.example;



import com.javadude.annotation.Access;

import com.javadude.annotation.Bean;

import com.javadude.annotation.Property;



@Bean(properties = {

        @Property(name="name"),

        @Property(name="age",type=int.class,reader=Access.NONE),

        @Property(name="address",type=Address.class,bound=true),

        @Property(name="ssn",writer=Access.PROTECTED)

      }

)    

public class Person extends PersonGen {

}

In this example we defined the following properties:

  • name - non-bound, read/write, String
  • age - non-bound, write-only, int
  • address - bound, read/write, Address
  • ssn - non-boud, read/write (writer is protected), String

If you wanted to define a bean that by default had all of its writers PROTECTED (only subclasses and other same-package classes can call it) by default, you could write the following:

package com.javadude.annotation.example;



import com.javadude.annotation.Access;

import com.javadude.annotation.Bean;

import com.javadude.annotation.Property;



@Bean(writer=Access.PROTECTED,

      properties = {

        @Property(name="name",writer=Access.PUBLIC),

        @Property(name="age",type=int.class),

        @Property(name="address",type=Address.class),

        @Property(name="ssn")

      }

)    

public class Person extends PersonGen {

}

In this example, the age, address, and ssn properties will have setter methods that are declared protected. The name property explicitly specifies that its setter method will be public.
