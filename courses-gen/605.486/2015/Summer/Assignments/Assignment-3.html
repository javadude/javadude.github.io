Assignment 3 - Manipulating Graphics

Requirements

Create a simple “Jewel Matcher” game.

  • The idea behind the game is to have a grid of random shapes. You can swap any two horizontally or vertically adjacent shapes if it allows you to make a “match”

  • Three or more adjacent shapes in a row or column of the same type are considered a “match”

  • If the user has made a move that creates one or more matches

    • Check for all possible matches - the user may have created a row and column or two rows or two columns etc. at the same time
    • Highlight the matching shapes for a moment (perhaps 300-500ms? find an amount that feels right to you)
    • Remove the matching shapes (blank them out for a moment)
      • Each matched shape counts as one point to keep things simple
    • Shift all shapes down (you can animate their movement if you want, but you do not need to - you can just redraw all shapes in their new positions)
    • Add new random shapes to fill in blanks

  • You must have four different shapes

    • Two must be defined using XML resources (circle and square, for example)
    • Two must be defined as custom shapes, similar to the triangle, but you cannot use triangle. Come up with two new shapes.

  • User interaction:

    • Tapping on a shape highlights all other shapes of the same type for a moment (again, 300-500ms?)
    • Dragging a shape moves it, like in the graph drawing example
    • If a shape is dragged onto another shape's grid area and let go
      • If no matches are possible, return the shape to its old position
      • If any matches are possible, swap the two shapes and do the “if the user has made a move that creates one or more matches” logic above
    • Tapping outside the grid “pauses” the game
      • Just blank the screen and write “Paused”
      • Tapping again unpauses
      • (This doesn't really do anything b/c we're not timing anything…)
    • Dragging outside the grid “shuffles” the pieces
      • To keep this simple, simply remove all pieces and re-add random pieces
      • Deduct 10 points when the user shuffles

  • The grid should be 8x8 on all devices

    • DO NOT assume the size of the grid cells; divide the screen into even squares, leaving some space around the grid

  • Do not worry about game states where making a match isn't possible (game over). (This can be a little complex to compute and I don't want to add in a lot of extra logic function)

  • Display the current score wherever you would like

  • Do not use third-party libraries for this assignment! You should be drawing your shapes and board manually, like the Module 11 example shows. (Though you don't have to use a “Thing” class; you can organize your code differently if you would like.)

Tips

To select random shapes:

    private Random random = new Random();
    ...
    random.nextInt(4); // returns 0-3 to choose which shape

Use an AsyncTask to do the “something matched” logic.

  • If you don't, you won't see updates to the user interface
  • See http://javadude.com/courses-static/605.486/2015/Summer/Code/AsyncTaskExample-v3.zip for an example of using AsyncTask
    • This example demonstrates performing logic inside the UI thread (bad) and inside an AsyncTask (good)
    • This example contains a good bit of comments describing how AsyncTask works
  • Note: Using this async task approach (with wait/notify), do not call get() from the UI thread on the async task or your app will deadlock (the UI would be waiting for the doInBackground to finish while it waits for the progress update to notify it…)
  • Note: You'll need to set a flag to track when you're actually running the task so you ignore user input while previous input is being processed

Submitting Your Assignment

To submit your assignment:

  1. Right-click your project(s) and choose Export…

  2. Under 'General' choose 'Archive File' and press Next

  3. Make sure the following options are checked:

    • Save in zip format
    • Compress the contents of the file
    • Create directory structure for files

  4. Choose the zip file to create in the To Archive File box.

    *The name of the zip file should be the same as your project name, followed by .zip

  5. Press Finish.

  6. Go to the Blackboard course website

  7. Click on the Assignments link on the left navigation bar

  8. Click on the Assignment 3 link

  9. Attach your submission and enter any notes

  10. Be sure to submit your assignment, not just save it!
