Module INSTANCEDATA - Saving Instance Data

Overview

When a device is rotated, the currently-displayed Activity is destroyed and recreated. In the process, any data that the user has entered will be lost unless it is explicitly saved.

In this module, we'll explore how you can save data when your Activity is rotated, and pass that data to other Activities. By implementing the Android Parcelable interface, we can attach objects to our Intents to transfer them to other Activities.

Objectives

By the end of this module, you will be able to:

  • Save instance data to prevent loss when the user rotates the device.
  • Create Parcelable object classes to be saved or transferred between Activities.
